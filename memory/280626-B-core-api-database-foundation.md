## Decisions / ADR notes
- Decision: Use UUID primary keys generated by the app for user/video/job/rendition/event rows.
- Reason: API-exposed video IDs are opaque and also appear in storage/HLS paths.
- Alternatives considered: BIGINT identity keys, which are simpler but expose sequence-like IDs in public route/storage contracts.
